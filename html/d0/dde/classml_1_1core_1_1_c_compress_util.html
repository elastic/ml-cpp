Detailed Description

Shrink wrap zlib calls.

DESCRIPTION:
Shrink wrap zlib calls.

IMPLEMENTATION DECISIONS:
Data can be added incrementally and then 'finished' to complete deflation or inflation.

This object retains in memory the entire compressed state so it not good for file read/write.

A single Z stream is used for the lifetime of the object, so each object can only work on one task at a time. In a multi-threaded application it would be best to create one object for each thread.
