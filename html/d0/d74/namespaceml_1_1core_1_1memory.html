Detailed Description

Core memory usage template class.

DESCRIPTION:
Core memory usage template class. Provides a method for determining the memory used by different ml classes and standard containers.

ML classes can declare a public member function:

std::size_t memoryUsage() const;

which should call memory::dynamicSize(t); on all its dynamic members.

For virtual hierarchies, the compiler can not determine the size of derived classes from the base pointer, so wherever the afore- mentioned memoryUsage() function is virtual, an associated function

std::size_t staticSize() const;
ml::core::memory::staticSize

should be declared, returning sizeof(*this).

template<typename K , typename V , typename H , typename P , typename A >
std::size_t ml::core::memory::bucketGroupOverhead ( const boost::unordered_map< K, V, H, P, A > & t)
constexpr

The overhead associated with the unordered container's bucket_groups. On average this is 4 bits per bucket. This is because the bucket_group structure consists of a 64 bit bitmask representing up to 64 buckets, plus 3 pointers, so a 32 byte structure stores the information for up to 64 buckets. See https://www.boost.org/doc/libs/1_83_0/libs/unordered/doc/html/unordered.html#structures_closed_addressing_containers for more detail.

Note
{ In practice we round up the bucket overhead to the next highest multiple of 4 * std::size_t}
