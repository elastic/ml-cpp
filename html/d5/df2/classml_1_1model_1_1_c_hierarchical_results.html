Detailed Description

Represents the bucket result of running a full analysis.

DESCRIPTION:
This wraps up the logic to build a hierarchy on top of a collection of simple search results. A simple search would, for example, be one clause of a model configuration file and has the command line syntax: 

  [partitionfield = w] function[(x)] [by y] [over z]

An abstract visitor pattern is implemented, with the intention of factoring out logic to, for example, output hierarchical results and aggregate the probabilities up the tree. Both bottom up depth and breadth first visiting strategies have been implemented.

IMPLEMENTATION DECISIONS:
This loosely implements a builder pattern: each simple search result is added and the intention is that all results are first added and then the hierarchical object is built (although buildHierarchy can be called repeatedly).

Most of the state of this class is held by reference and could become invalid if it is kept longer than to output a single result. This is to minimize the amount of state that needs to be copied when outputting results (to minimize both runtime and transient memory usage).

Add a simple search result.

The general syntax for a simple search is 

  [partitionfield = w] function[(x)] [by y] [over z]

Examples include:

  1. count
  2. rare by x
  3. partitionfield = x mean(y)
  4. min(x) over z
  5. partitionfield = x dc(y) over z
  6. partitionfield = w max(x) by y over z

If a given search doesn't have a field pass the empty string.

Parameters
[in]detectorAn identifier of the detector generating this result.
[in]isPopulationTrue if this is a population result and false otherwise.
[in]functionNameThe name of the function of the model's search.
[in]functionThe function of the model's search.
[in]partitionFieldNameThe partition field name or empty.
[in]partitionFieldValueThe partition field value or empty.
[in]personFieldNameThe over field name or empty.
[in]personFieldValueThe over field value or empty.
[in]valueFieldNameThe name of the field containing the metric value.
[out]annotatedProbabilityA struct containing the probability, the smallest attribute probabilities, the influencers, and any extra descriptive data
[in]modelThe model which generated the result.
Note
Values which are passed by non-constant reference are swapped in to place.

◆ buildHierarchy()

void ml::model::CHierarchicalResults::buildHierarchy ( )

Build a hierarchy from the current flat node list using the default aggregation rules.

The aggregation rules in priority order are:

  1. Only aggregate searches with the same partition field name and value.
  2. Subject to 1, aggregate searches with the same person field name and value: this is the by field name and value if no over field is specified otherwise it is the over field name name and value.
