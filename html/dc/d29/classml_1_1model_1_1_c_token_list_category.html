Detailed Description

The data associated with this ML category.

DESCRIPTION:
The data associated with this ML category

IMPLEMENTATION DECISIONS:
Base the category on a string

Token IDs are stored rather than the actual string tokens. This improves performance. The CTokenListDataCategorizer object that created this object knows the mappings between the token IDs and string tokens.

template<typename PAIR_CONTAINER >
bool ml::model::CTokenListCategory::isMissingCommonTokenWeightZero ( const PAIR_CONTAINER & uniqueTokenIds) const
inline

Is the weight of tokens in the provided container that are missing from this category's common unique tokens equal to zero? It is possible to test: if (category.missingCommonTokenWeight(uniqueTokenIds) == 0) instead of calling this method. However, this method is much faster as it can return false as soon as a mismatch occurs.

Parameters
uniqueTokenIdsA container of pairs where the first element is a token ID and the container is sorted into ascending token ID order.

◆ orderedCommonTokenBounds()

CTokenListCategory::TSizeSizePr ml::model::CTokenListCategory::orderedCommonTokenBounds ( ) const
Returns
A pair of indices indicating the beginning and end of the common ordered tokens within the base tokens. Importantly, the tokens within the range may not all be common across the category. The consumer of these bounds must check whether each base token in the range is common before using it for any purpose that relies on commonality (for example creating a reverse search).
