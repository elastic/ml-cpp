Detailed Description

Implementation for a multimodal prior distribution.

DESCRIPTION:
This is used to model a variable for which we expect there to be distinct modes, which can be modeled accurately by any of our basic single mode distributions.

A separate mechanism is provided to identify the clusters in the data corresponding to distinct modes so that different methods for identifying clusters can be used.

All prior distributions implement a process whereby they relax back to the non-informative over some period without update (see propagateForwardsByTime). The rate at which they relax is controlled by the decay factor supplied to the constructor.

IMPLEMENTATION DECISIONS:
All priors are derived from CPrior which defines the contract that is used by composite priors. This allows us to select the most appropriate model for the data when using one-of-n composition (see COneOfNPrior). From a design point of view this is the composite pattern.
