Detailed Description

Manages recording the progress of a loop.

DESCRIPTION:
It is generally bad practice to record progress every iteration of a loop since:

  1. This level of granularity of progress reporting is not needed
  2. In parallel code this can cause a lot of contention on some shared resource.

This class manages breaking the progress reporting of a loop into a fixed number of steps.

Example:

double progress{0.0};
CLoopProgess loopProgess{n, [&progess](double p) { progress += p}};
for (std::size_t i = 0; i < n; ++i, loopProgess.increment()) {
...
}
std::cout << progress << std::endl;

Output: 1

IMPLEMENTATION:\N The number of steps chosen to be a power of 2 so that the progress per step is exactly representable as an IEEE floating point type. This means in normal usage the total progress will be exactly 1.0 at the end of a loop.
