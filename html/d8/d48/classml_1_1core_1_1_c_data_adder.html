Detailed Description

Abstract interface for data adder.

DESCRIPTION:
Contains methods that require data be known in advance, plus a method that returns a stream to which the data to be persisted can be written. This latter method is obviously much more memory-efficient in cases where it is a viable option.

IMPLEMENTATION DECISIONS:
There's an assumption that persisted state will be saved to a data store that can retrieve based on a single ID.

Clients that get a stream using addStreamed() must call this method one they've finished sending data to the stream. They should set force to true when the very last stream is complete, in case the persister needs to close off some sort of cached data structure.
