Detailed Description

Reflection for CTimeSeriesMultibucketFeature sub-classes.

DESCRIPTION:
Encapsulate the conversion of arbitrary CTimeSeriesMultibucketFeature sub-classes to/from textual state. In particular, the field name associated with type of feature is then in one file.

IMPLEMENTATION DECISIONS:
The serialisation format must be a hierarchical format that supports name/value pairs where the value may be a nested set of name/value pairs. Text format is used to make it easier to provide backwards compatibility in the future as the classes evolve.
