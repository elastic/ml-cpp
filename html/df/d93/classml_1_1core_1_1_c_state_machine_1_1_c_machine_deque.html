Detailed Description

A custom paired down std::deque like container.

DESCRIPTION:
We have rather specific requirements for this container:

  1. It must be (as) random access (as possible),
  2. It must be possible for push_back to occur concurrently with lookup of an existing item in the container.

IMPLEMENTATION DECISIONS:
With std::deque implementations any invocation of operator[] can fail if there is a concurrent push_back. The code using this class ensures that it only ever asks for an element which already exists in the container at the start of push_back. This is possible safely with a std::list. However, since this also needs to be random access, using a vanilla std::list, which has complexity for accessing the item and poor locality of reference, is not suitable. Instead we prefer to use a list of preallocated std::vectors, on which it is also safe to call push_back, for our use case, provided doing so doesn't cause them to reallocate.
