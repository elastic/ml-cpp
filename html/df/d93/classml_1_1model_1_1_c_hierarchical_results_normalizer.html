Detailed Description

A collection of normalizers for some hierarchical results.

DESCRIPTION:
Manages the normalization of a collection of hierarchical results. Each distinct partition and "named system" or population has its own normalizer in addition the unique "simple search" results, belonging to the same named partition and population, have a normalizer. Finally the bucket results have a normalizer. For example, if someone were to configure the following searches:

  partitionfield=region sum(bytes) by host
  partitionfield=region count by host
  partitionfield=region sum(bytes) over client
  partitionfield=host rare by process

There would be nine normalizers in total:

  1. One normalizer for the bucket,
  2. One normalizer for count and sum(bytes) probabilities for system host and partition region,
  3. One normalizer for system host and partition region,
  4. Two (identical) normalizers for client and partition region,
  5. One normalizer for partition region,
  6. Two (identical) normalizers for process and partition host
  7. One normalizer for partition host.

Note that the redundancy, for client and process fields, is because each of these has one search; however, we still create normalizers at both the "simple search" and "named system" or population level. It is impossible for us to determine there is redundancy observing results objects because we don't know if some results are not present for the bucket, because a field value wasn't present in the bucket. The wasted memory in creating a small number of redundant normalizers is negligible.
