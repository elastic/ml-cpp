Detailed Description

Interface for a multivariate prior distribution function.

DESCRIPTION:
Abstract interface for implementing multivariate prior distribution functions for various classes of likelihood functions.

This exists to support a one-of-n prior distribution which comprises a weighted selection of basic likelihood functions and is implemented using the composite pattern.

Compute the bivariate prior marginalizing over the variables marginalize and conditioning on the variables condition.

Parameters
[in]marginalizeThe variables to marginalize out.
[in]conditionThe variables to condition on.
Warning
The caller owns the result.
Note
The variables are passed by the index of their dimension which must therefore be in range.
It is assumed that the variables are in sorted order.
The caller must specify dimension - 2 variables between marginalize and condition so the resulting distribution is univariate.

◆ printMarginalLikelihoodFunction()

std::string ml::maths::common::CMultivariatePrior::printMarginalLikelihoodFunction ( std::size_t x,
std::size_t y ) const

Print a projection of the marginal likelihood function onto the specified coordinates.

The format is as follows:

x = [x1 x2 .... xn ];
y = [y1 y2 .... yn ];
likelihood = [L(x1, y1) L(x1, y2) ... L(x1, yn)
L(x2, y1) L(x2, y2) ... L(x2, yn)
. . .
. . .
. . .
L(xn, y1) L(xn, y2) ... L(xn, yn) ];

i.e. domain values are space separated on the first and subsequent line(s) as appropriate and the density function evaluated at those values are space separated on the next line and subsequent lines as appropriate.

◆ sampleMarginalLikelihood()

virtual void ml::maths::common::CMultivariatePrior::sampleMarginalLikelihood ( std::size_t numberSamples,
TDouble10Vec1Vec & samples ) const
pure virtual

Sample the marginal likelihood function.

The marginal likelihood functions are sampled in quantile intervals of the generalized cumulative density function, specifically intervals between contours of constant probability density.

The idea is to capture a set of samples that accurately and efficiently represent the information in the prior. Random sampling (although it has nice asymptotic properties) doesn't fulfill the second requirement: typically requiring many more samples than sampling in quantile intervals to capture the same amount of information.

This is to allow us to transform one prior distribution into another completely generically and relatively efficiently, by updating the target prior with these samples. As such the prior needs to maintain a count of the number of samples to date so that it isn't over sampled.

Parameters
[in]numberSamplesThe number of samples required.
[out]samplesFilled in with samples from the prior.
Note
numberSamples is truncated to the number of samples received.
