Detailed Description

The Ml anomaly detector.

DESCRIPTION:
Take a stream of input records and read those records according to the given field config.

IMPLEMENTATION DECISIONS:
Input must be in ascending time order.

The output format is so complex that this class requires its output handler to be a CJsonOutputWriter rather than a writer for an arbitrary format

◆ ERestoreStateStatus

enum ml::api::CAnomalyJob::ERestoreStateStatus

Enum represents the result of persisted Model restoration Possible states are:

  1. IncorrectVersion: The version of the stored model state does not match the anomaly detector version.
  2. UnexpectedTag: State is malformed or could not be parsed correctly
  3. MemoryLimitReached: The detector could not be allocated becasuse it would violate the memory usage restrictions
  4. NotRestoredToTime: The detector was not restored to the requested time
  5. Success:
  6. Failure:

◆ m_InitialLastFinalisedBucketEndTime

core_t::TTime ml::api::CAnomalyJob::m_InitialLastFinalisedBucketEndTime {0}
private

Introduced in version 8.6 The initial value of the end time of the last bucket out of latency window we've seen, i.e. this member records the first non-zero value of m_LastFinalisedBucketEndTime and then never changes. When restoring jobs that ran successfully for many buckets before being persisted by a version earlier than 8.6 this member will always have a value of 0, therefore it is crucial that this member is never assumed to be non-zero and should only be used for its intended purpose of aiding in detecting an incomplete initial bucket after state restoration.
