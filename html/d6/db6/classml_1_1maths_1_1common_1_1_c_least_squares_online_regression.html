Detailed Description

template<std::size_t N_, typename T = CFloatStorage, bool R_2 = false>
class ml::maths::common::CLeastSquaresOnlineRegression< N_, T, R_2 >

DESCRIPTION:
A very lightweight online weighted least squares regression to fit degree N polynomials to a collection of points , i.e. to find the s.t. the weighted sum of the square residuals is minimized. Formally, we are looking for defined as

Here, denotes the design matrix and for a polynomial takes the form . This is solved using the Moore-Penrose pseudo-inverse.

We are able to maintain sufficient statistics to construct and also the components of the vector online.

IMPLEMENTATION DECISIONS:
This uses float storage and requires floats where is the polynomial order. In total this therefore uses bytes.

Note that this constructs the Gramian of the design matrix when computing the least squares solution. This is because holding sufficient statistics for constructing this matrix is the most space efficient representation to compute online. However, the condition of this matrix is the square of the condition of the design matrix and so this approach doesn't have good numerics.

A much more robust scheme is to use incremental QR factorization and for large problems that approach should be used in preference. However, much can be done by using an affine transformation of to improve the numerics of this approach and the intention is that it is used for the case where is small and space is at a premium.

Template Parameters
N_The degree of the polynomial.
TThe storage type used for the statistics. Note that since we can't avoid losing precision in our formulation be careful using single precision if the polynomial degree is high.
R_2If this is true you can additionally compute R^2 and the residual statistics, but it adds sizeof(T) to the memory usage.

In particular, one will typically use rather than directly, since can be adjusted to improve the condition of the Gramian.

If this is running online, then as x increases one wants to allow the shift to increase. This function computes the impact of a change in on the stored statistics.
