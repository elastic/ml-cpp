Detailed Description

Interface for a prior distribution function.

DESCRIPTION:
Abstract interface for implementing prior distribution functions for various classes of likelihood functions.

This exists to support a one-of-n prior distribution which comprises a weighted selection of basic likelihood functions and is implemented using the composite pattern.

virtual bool ml::maths::common::CPrior::minusLogJointCdf ( const TDouble1Vec & samples,
const TDoubleWeightsAry1Vec & weights,
double & lowerBound,
double & upperBound ) const
pure virtual

Calculate minus the log of the joint c.d.f. of the marginal likelihood for a collection of independent samples from the variable.

Parameters
[in]samplesA collection of samples of the variable.
[in]weightsThe weights of each sample in samples.
[out]lowerBoundFilled in with a lower bound for where is the c.d.f. and are the samples.
[out]upperBoundFilled in with a upper bound for where is the c.d.f. and are the samples.
Note
The samples are assumed to be independent.
In general, lowerBound equals upperBound. However, in some cases insufficient data is held to exactly compute the c.d.f. in which case the we use sharp upper and lower bounds.
Warning
The variance scales must be in the range , i.e. a value of zero is not well defined and a value of infinity is not well handled. (Very large values are handled though.)

◆ sampleMarginalLikelihood()

virtual void ml::maths::common::CPrior::sampleMarginalLikelihood ( std::size_t numberSamples,
TDouble1Vec & samples ) const
pure virtual

Sample the marginal likelihood function.

The marginal likelihood functions are sampled in quantile intervals. The idea is to capture a set of samples that accurately and efficiently represent the information in the prior. Random sampling (although it has nice asymptotic properties) doesn't fulfill the second requirement: typically requiring many more samples than sampling in quantile intervals to capture the same amount of information.

This is to allow us to transform one prior distribution into another completely generically and relatively efficiently, by updating the target prior with these samples. As such the prior needs to maintain a count of the number of samples to date so that it isn't over sampled.

Parameters
[in]numberSamplesThe number of samples required.
[out]samplesFilled in with samples from the prior.
Note
numberSamples is truncated to the number of samples received.
