Detailed Description

Parses a complete specification for running a core::CDataFrame analysis and supports launching that analysis on a specified frame object.

DESCRIPTION:
This manages extracting all configuration for a particular analysis from a JSON header which is passed to the data_frame_analyzer command before any data. This creates and owns an analysis runner object which is also configured by the header. The analysis is run asynchronously via the CDataFrameAnalysisSpecification::run method which returns a handle to the runner to retrieve progress, errors and other performance statistics.

Initialize from a JSON object.

The specification has the following expected form: { "job_id": <string>, "rows": <integer>, "cols": <integer>, "memory_limit": <integer>, "threads": <integer>, "temp_dir": <string>, "results_field": <string>, "categorical_fields": [<string>], "disk_usage_allowed": <boolean>, "analysis": { "name": <string>, "parameters": <object> } }

Parameters
[in]jsonSpecificationThe specification as a JSON object.
Note
The analysis name must be one of the supported analysis types.
All constraints must be positive.
The parameters, if any, must be consistent for the analysis type.
If this fails the state is set to bad and the analysis will not run.
temp_dir Is a directory which can be used to store the data frame out-of-core if we can't meet the memory constraint for the analysis without partitioning.
Parameters
persisterSupplierShared pointer to the CDataAdder instance.
