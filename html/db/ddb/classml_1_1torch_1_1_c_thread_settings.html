void ml::torch::CThreadSettings::numAllocations ( std::int64_t numAllocations)

Set the number of allocations, but constraining the value given the current values of max threads and threads per allocation. The way the number of allocations is constrained is different to how it's done in validateThreadingParameters(), because we cannot change threads per allocation after it's initially set.

void ml::torch::CThreadSettings::validateThreadingParameters ( std::int64_t & maxThreads,
std::int64_t & numThreadsPerAllocation,
std::int64_t & numAllocations )
static

Validate the max threads, number of threads per allocation and number of allocations to ensure that CPU is not overcommitted, as this leads to worse performance than running with fewer threads. This method prefers to constrain threads per allocation instead of number of allocations.
